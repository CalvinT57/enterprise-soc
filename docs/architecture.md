# Project Sentinel
## Enterprise Security Operations Center

---

## Executive Summary

Project Sentinel is a simulated enterprise Security Operations Center (SOC) built to emulate a real-world corporate environment at Apex Manufacturing.

The goal of this project is to design, deploy, secure, monitor, and defend an enterprise network using industry-standard technologies while documenting the entire engineering process.

The environment will support:

- Enterprise Active Directory
- Windows endpoints
- Linux infrastructure
- Network intrusion detection
- Security monitoring
- Threat hunting
- Incident response
- Detection engineering
- Security automation
- Cloud integration

---

# Business Scenario

Apex Manufacturing is a mid-sized manufacturing company with approximately 250 employees.

The organization relies on Windows Active Directory, Linux web servers, cloud services, and remote users to conduct business operations.

Due to increasing ransomware attacks targeting manufacturing organizations, the company has decided to modernize its security operations.

Project Sentinel was created to improve:

- Visibility
- Threat Detection
- Incident Response
- Asset Monitoring
- Security Automation

---

# Security Objectives

The Enterprise SOC must provide:

- Centralized logging
- Endpoint visibility
- Network visibility
- Threat detection
- Incident investigations
- MITRE ATT&CK mapping
- Incident reporting
- Automated alerting

---

# Success Criteria

The project will be considered successful when:

- Enterprise infrastructure is deployed
- Security telemetry is centralized
- Attack simulations generate detections
- Incidents are investigated
- Reports are documented
- Detection rules are created
- Security workflows are automated
